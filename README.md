# Прогнозирование пассажиропотока (GAT vs базлайны)

Global-forecasting пассажиропотока московского транспорта (метро / НГПТ / МЦД), ~1500 объектов, часовые ряды, **очень малый объём истории (~168 ч ≈ 1 неделя)**. Метрики MAE / SMAPE / WAPE считаются по дневным часам 6:00–23:00, оценка — rolling-backtest.

- **Данные:** агрегаты транзакций валидаций (`PASS_ALL…csv` + справочники), качаются с Google Drive скриптом подготовки (`--prepare`), разворачиваются в часовой `pivot` top-1500 объектов.
- **Целевая модель:** EgoGAT (`src/models/gat/`) — батчинг по (объект, окно), residual-граф top-k соседей, темпоральный энкодер + per-step GRU-декодер, seasonal-naive якорь.
- **Базлайны для сравнения:** TFT, TCN, N-BEATS (`src/models/{tft,tcn,nbeats}/`) и классические профили/ETS-ансамбли (`src/baseline_models.py`).

> Особенность задачи: **данных мало** (~168 ч ≈ 1 неделя), сильная суточная и недельная сезонность, объекты скоррелированы (пересадки, узлы). Поэтому в обучении используется режим `real_plus_synth` (реальная неделя + синтетическая история).

## Цель проекта

Проект решается в условиях **крайне малого объёма данных**, поэтому цель смещена с «достичь минимальной ошибки любой ценой» на работу в low-data regime:

- **Стабильные и адекватные метрики на малой выборке.** Приоритет — устойчивость прогноза по фолдам и по объектам, а не единичное рекордное значение метрики. Поэтому модели сравниваются по «зрелым» метрикам (среднее по последним 2 фолдам), а не по лучшему фолду.
- **Поиск методов, устойчивых к малому объёму данных.** Систематически проверяется, какие подходы реально помогают при недельной истории (наивные сезонные профили, классические модели, глубокие сети, графовые идеи, синтетика), а какие переобучаются или нестабильны (см. абляцию A/B/C/D и надстройки).
- **Практическая применимость.** Решение должно давать разумный прогноз сразу на ~1500 объектах в рамках ограничений по ресурсам

В этой рамке сравниваются классические бейзлайны, глубокие модели (TFT/TCN/N-BEATS) и графовая модель EgoGAT с идеями из статей; рабочий ориентир — превзойти TFT по MAE/WAPE на обоих горизонтах при сохранении стабильности.

## Суть работы

1. Сырые транзакции агрегируются до часового пассажиропотока по объектам (`object_id` = `ST_*` станции / `RT_*` маршруты / `PL_*` площадки) и разворачиваются в матрицу `время × объект` (top-1500 по суммарному потоку).
2. Объекты кластеризуются по суточным профилям (`final_clusters.csv`) — кластеры используются как статический признак узла и для иерархии.
3. Поскольку реальной истории мало, генерируется **синтетическая** предыстория (`src/synthesis.py`) и приклеивается перед реальной неделей (`real_plus_synth`).
4. Модели обучаются и оцениваются в **rolling-backtest** по фолдам (`src/backtest.py`); метрики — MAE / RMSE / MAPE / SMAPE / WAPE на дневных часах.
5. Для GAT реализован ряд идей из статей (каждая включается флагом CLI); **замеры в таблице результатов** покрывают абляцию A/B/C/D и надстройки D + SAN, D + TimesNet — то есть методы adaptive embeddings, adaptive adjacency, SAN и TimesNet.
6. Результаты прогонов сохраняются в `eda_output/` (`gat_*_{H}h.csv`) и визуализируются **отчётными ноутбуками** (`notebooks/gat_report_*`); сводные метрики собираются в `metrics_tables.md`.

## Архитектура GAT (EgoGAT)

Код: `src/models/gat/pipeline.py` (модель, граф, train/predict/backtest), `run.py` (CLI).

- **Батчинг по (объект, окно)** — как у TFT, вместо full-graph батчинга; даёт тысячи i.i.d. шагов оптимизатора.
- **Residual-граф** (`build_graph`): из каждого ряда вычитается профиль (час × будни/выходной), ego-граф строится по корреляции **остатков** (top-k соседей), что отражает совместные отклонения (пересадки/узлы), а не общий суточный цикл.
- **Темпоральный энкодер** (dilated causal conv) сохраняет все T состояний; **per-step GRU-декодер** + temporal cross-attention на свою историю + gated spatial cross-attention на состояния соседей.
- **Выход** = seasonal-naive якорь + поправка; ночные часы даунвейтятся в лоссе (`--night-weight`).
- **FiLM-кондиционирование** на статические признаки узла (`src/static_features.py`).

### Идеи из статей (флаги CLI)

Ниже — только идеи, для методов которых есть замеры в таблице результатов (`metrics_tables.md`). Нумерация идей — по `GAT_WORK_SUMMARY.md`. `--no-adaptive` отключает обе adaptive-идеи (= baseline-EgoGAT, конфиг A).

| # | Идея (статья) | Флаг | Конфиг с замером | Как использовано |
|---|---|---|---|---|
| 1 | **Adaptive embeddings** (STAEformer, CIKM’23) | вкл по умолч., выкл `--no-adaptive-embed` | B, D | Обучаемые эмбеддинги узлов (идентичность объекта). |
| 2 | **Adaptive adjacency** (Graph WaveNet IJCAI’19 / AGCRN NeurIPS’20) | выкл `--no-adaptive-adj` | C, D | Обучаемое ребро = ⟨e_i, e_j⟩ поверх residual-графа. |
| 4 | **SAN** slice-нормировка (NeurIPS’23) | `--san`, `--san-period` | D + SAN | Послайсовая нормировка + предсказание статистик слайса. |
| 5 | **TimesNet** 2D-мультипериодичность (ICLR’23, arXiv 2210.02186) | `--timesnet`, `--timesnet-blocks`, `--timesnet-k` | D + TimesNet | Замена dilated-энкодера на 2D-моделирование периодов (FFT). |

Дополнительные инструменты: `--ensemble K` (сид-ансамбль), `--save-predictions`/`blend.py` (бленд прогонов).

## Данные и пайплайн подготовки

Подготовку выполняет `src/data_prep.py` (вызывается через `--prepare` в `run.py`):

1. **`download_and_extract`** — качает архив с Google Drive (`gdown`, id `1resyBioT2TJTZdLVfuEZdk8PvJAbK5GM`) в `data.zip`, распаковывает его и вложенный `Метро НБС/pass_10-160324.zip` в `data/`.
2. **`create_hourly_parquet`** — потоково (чанки) читает `PASS_ALL…csv`, фильтрует валидные входы (`VALIDATION_MODE=1`, `VALIDATION_RESULT=1`, `IS_FAIL=0`), агрегирует до `pax` по (час, площадка/маршрут), джойнит справочники (`REF_PSG_PLACES`, `REF_TRANSPORT_TYPE`, `REF_TRANSPORT_WAY`), размечает категорию транспорта (Metro/NGPT/MCD/Other), выходные и праздники (`HOLIDAYS_MD`) → `eda_output/hourly.parquet`.
3. **`create_object_hourly_parquet`** — агрегирует до уровня `object_id` → `eda_output/object_hourly.parquet`.
4. **`make_pivot`** — разворачивает в матрицу `date_hour × object_id`, реиндексирует на полный часовой диапазон, оставляет top-N (по умолчанию **1500**) объектов по суммарному потоку.
5. **`run_clustering`** (`src/clustering.py`) — кластеризует объекты по суточным профилям → `eda_output/final_clusters.csv`.

Артефакты подготовки и результаты прогонов лежат в `eda_output/`.

### Отчётные ноутбуки GAT

Для каждой конфигурации — два ноутбука (h=1, h=24) одинаковой структуры. Они **читают** сохранённые `eda_output/gat_{summary,results,history}_{H}h.csv` и строят: итоговые метрики, метрики по «зрелым» фолдам (последние 2), кривую обучения, метрики по фолдам, распределение ошибки по объектам, ошибку vs объём, топ объектов и текстовый вывод. ## Обучение

Запуск GAT, команды одной из лучших моделей:

```bash
# h=24: D + TimesNet
!python -m src.models.gat.run --prepare --horizons 24 --protocol rolling \
  --train-modes real_plus_synth --synth-days 45 --num-workers 4 --epochs 11 --timesnet

# h=1: D + TimesNet (blocks 1, чтобы влезть в лимит времени)
!python -m src.models.gat.run --prepare --horizons 1 --protocol rolling --step-1h 6 \
  --train-modes real_plus_synth --synth-days 45 --num-workers 4 --epochs 11 \
  --max-folds 3 --timesnet --timesnet-blocks 1
```

Конфигурации абляции:

- **A — baseline:** добавить `--no-adaptive` (обе adaptive-идеи выключены).
- **B — только embeddings:** `--no-adaptive-adj`.
- **C — только adjacency:** `--no-adaptive-embed`.
- **D — обе идеи:** без флагов энкодера.
- **D + SAN:** D + `--san`. **D + TimesNet:** D + `--timesnet --timesnet-blocks 1`.

Базовые модели запускаются по тому же паттерну: `python -m src.models.{tft,tcn,nbeats}.run ...`.

## Бейзлайн-модели (классические)

Ноутбук `notebooks/baseline-metro_full.ipynb` оценивает набор простых, **не требующих обучения** моделей — нижнюю планку, относительно которой меряются глубокие модели. Это варианты «прогноз по истории/профилю», устойчивые к малому объёму данных, потому что в них почти нет обучаемых параметров.

Краткое описание моделей:

- **Seasonal Naive / Seasonal-24** — повтор значения «сутки назад в этот же час» (период 24 ч).
- **Naive-1** — повтор последнего наблюдённого часа.
- **Mean-3** — среднее за последние 3 часа.
- **Mean Profile** — усреднённый суточный профиль по всем дням истории.
- **Weighted Profile** — суточный профиль с экспоненциальными весами (свежие дни весомее).
- **Same-Type Day** — профиль, усреднённый отдельно по будням и выходным (берётся тип дня прогноза).
- **Holiday-Aware** — `Same-Type Day` с подменой праздничного дня на воскресный профиль.
- **ETS (`ImprovedETS`)** — Холт-Уинтерс (аддитивная/мультипликативная сезонность) с выбором лучшего кандидата по внутренней валидации.
- **k-NN minimal (`MinimalKNN`)** — kNN-регрессия по признакам «час (sin/cos) + лаг 24 ч».
- **Clean Ensemble** — взвешенное (по обратной валидационной MAE) усреднение seasonal / same-type / holiday-aware / weighted.

Подробное описание — в [`docs/methods/classical_baselines.md`](docs/methods/classical_baselines.md).

> ⚠️ **Протокол отличается** от раздела «Результаты» ниже: метрики считаются **по всем 24 часам** (не только дневным 6:00–23:00) и по другой схеме фолдов (24 ч — 5 фолдов; 1 ч — 120 фолдов). Поэтому абсолютные значения **не сопоставимы напрямую** с метриками GAT/TFT из `metrics_tables.md` — таблицы ниже представляют отдельную, самостоятельную нижнюю планку. MAE приведён и средний, и медианный (среднее раздувают крупные объекты).

## Источники

Указаны только статьи, методы которых имеют замеры в таблице результатов (`metrics_tables.md`).

1. **STAEformer** (CIKM’23) — adaptive node embeddings. Идея 1 (конфиги B, D): обучаемые эмбеддинги узлов кодируют идентичность объекта (`use_adaptive_embed`).
2. **Graph WaveNet** (IJCAI’19) и **AGCRN** (NeurIPS’20) — обучаемая (adaptive) матрица смежности. Идея 2 (конфиги C, D): ребро как скалярное произведение эмбеддингов узлов поверх residual-графа (`use_adaptive_adj`).
3. **SAN — Slice Adaptive Normalization** (NeurIPS’23) — послайсовая нормировка с предсказанием статистик слайса. Идея 4 (конфиг D + SAN): борьба с нестационарностью (`--san`).
4. **TimesNet** (ICLR’23, arXiv 2210.02186) — 2D-моделирование мультипериодичности через FFT-периоды. Идея 5 (конфиг D + TimesNet): альтернативный темпоральный энкодер (`--timesnet`).

**Данные:** агрегаты валидаций пассажиров московского транспорта (`PASS_ALL…csv` и справочники `REF_*`, `GDS_GOODS`, `V_PCR_CONTRACTOR`), скачиваются с Google Drive скриптом `src/data_prep.py`.
