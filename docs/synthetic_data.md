# Синтетические данные

Код: [`src/synthesis.py`](../src/synthesis.py). Режим обучения с синтетикой — `real_plus_synth` (флаги `--synth-days`, `--synthetic-window-stride` в CLI моделей).

## Зачем нужна синтетика

Реальной истории — около **168 часов (1 неделя)**. Этого мало, чтобы глубокие модели (GAT/TFT/TCN/N-BEATS) увидели достаточно примеров суточной и недельной сезонности и не переобучились. Синтетика **наращивает обучающую выборку**, добавляя перед реальной неделей правдоподобную предысторию из тех же объектов, с теми же суточными формами, но с контролируемой вариативностью. Это решает проблему дефицита обучающих окон: модель получает тысячи дополнительных (объект × окно)-примеров, сохраняя статистические свойства реальных рядов.

## Как генерируются данные (высокий уровень)

Метод (из `save_generation_config`): **day-block bootstrap + global/cluster magnitude warping + Poisson-шум**. Генерация идёт **внутри каждого train-фолда** (`synthesize_from_train`) и приклеивается по времени **перед** реальными данными (`prepend_synthetic_week`).

Пошагово, для каждого синтетического дня:

1. **Day-block bootstrap.** История нарезается на суточные блоки 24×N по дню недели (`build_day_blocks`). Для целевого дня случайно выбирается реальный блок **того же типа дня** (будни/выходной), для праздника — воскресный блок (`_choose_foundation_block`). Так сохраняется реальная суточная форма.
2. **Global magnitude warp.** Весь дневной профиль умножается на плавную случайную кривую (кубический сплайн по 4 узлам), моделирующую общегородское «громче/тише» в разные часы (`generate_global_warp`).
3. **Cluster warp** (опц.). Дополнительная плавная поправка на уровне кластера объектов (`generate_cluster_warp`) — разные районы/типы станций ведут себя по-разному.
4. **Аномалии** (опц.). С вероятностью 10 % в случайный дневной час у случайного кластера вставляется всплеск (×1.4–1.8 + аддитивная добавка) — имитация событий.
5. **Тренды** (опц.). Медленный линейный дрейф уровня (глобальный + кластерный + пообъектный шум), нарастающий с номером дня.
6. **Poisson-шум.** Итоговая интенсивность пропускается через `np.random.poisson` — это даёт счётный (целочисленный) характер пассажиропотока и реалистичную дисперсию у малых значений.

Праздничные дни дополнительно приглушаются (×0.75–0.90).

## Валидация синтетики

`validate_synthetic` сравнивает синтетику с реальным хвостом по нескольким критериям и выставляет флаг `ready_for_training`:

| Метрика | Что проверяет | Порог годности |
|---|---|---|
| `acf_similarity` | сходство автокорреляции (временная структура) | > 0.80 |
| `profile_corr` | корреляция среднесуточных профилей | > 0.85 |
| `quantile_ratio` | сопоставимость масштабов (квантили 0.25/0.5/0.99) | 0.65–1.50 |
| `corr_mae` | сходство матрицы межобъектных корреляций | < 0.20 |

## Ключевые параметры генерации

| Параметр | Значение по умолчанию | Описание |
|---|---|---|
| `gen_days` | 7 (для `prepend_synthetic_week`) / 30 | Сколько дней синтетики сгенерировать. |
| `--synth-days` (CLI) | 45 (GAT/TFT), 30 (TCN/N-BEATS) | Дней синтетики в режиме `real_plus_synth`. |
| `seed` | 42 | Сид генератора (детерминизм). |
| `global_warp_sigma` | 0.06 (буд.) / 0.12 (вых.) | Амплитуда глобального warp. |
| `cluster_warp_sigma` | 0.04 (буд.) / 0.08 (вых.) | Амплитуда кластерного warp. |
| `anomaly_probability` | 0.10 | Вероятность всплеска в день. |
| тренды | глоб. ±0.04, кластер ±0.03, объект ~N(0, 0.01) | Диапазоны линейного дрейфа. |
| праздничный множитель | 0.75–0.90 | Приглушение праздничных дней. |
| `get_synth_days` | base 30, max_ratio 3.0, cap 90 | Адаптивный выбор числа дней от длины train. |

## Риски и ограничения

- **Смещение распределения (distribution shift).** Синтетика — это бутстрап реальных блоков с warp/шумом; она **не привносит новой информации**, а лишь переупаковывает имеющуюся неделю. Если реальная неделя нерепрезентативна (нет праздников, особых событий), синтетика унаследует это смещение. На текущих данных праздников не было — праздничная логика не проверена на реальных праздниках.
- **Утечка (leakage).** Критично: синтетика генерируется **только внутри train-фолда**. Генерация на полном ряде привела бы к утечке будущего в обучение. Это инвариант, который нельзя нарушать.
- **Переобучение на синтетике.** Слишком много синтетики или слишком сильный warp могут «увести» модель в синтетический режим. Поэтому объём ограничен (`--synth-days`), а warp/тренды держатся малыми; валидация (`validate_synthetic`) отбраковывает расхождения по профилю/корреляциям.
- **Гиперупрощённая структура аномалий.** Всплески — простые мультипликативные пики на кластер; реальные события (погода, перекрытия) сложнее и здесь не моделируются.
- **Poisson-предположение.** Счётность задаётся пуассоновским шумом (дисперсия = среднему); реальные пики могут быть сверхдисперсными, что синтетика недооценивает.
