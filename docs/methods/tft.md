# TFT — Temporal Fusion Transformer (бейзлайн)

Код: [`src/models/tft/`](../../src/models/tft/) (`pipeline.py` + `run.py`). Это **главный ориентир**: цель GAT — превзойти TFT по MAE/WAPE. Замеры: строки `TFT` в `metrics_tables.md`.

## Идея метода

TFT — attention-архитектура для многогоризонтного прогнозирования, сочетающая LSTM-кодирование локальной динамики с self-attention по горизонту и механизмами отбора признаков (variable selection) и гейтингом. Поддерживает прошлые и **будущие известные** ковариаты (время суток, день недели).

## Как работает (высокий уровень)

- LSTM-энкодер/декодер обрабатывает прошлое окно и известное будущее.
- Multi-head attention связывает шаги горизонта с релевантной историей.
- Gating/variable-selection отсеивает неинформативные признаки.
- В этом проекте обучается глобально (общие веса на все объекты), батч по (объект, окно), как и GAT.

## Применимость к малым данным

**Умеренно сильна.** В low-data regime TFT — крепкий бейзлайн: на h=24 он по зрелым метрикам лучше всех GAT-конфигов, кроме D и его надстроек; на h=1 уступает графовым моделям по MAE/WAPE. Глобальное обучение и будущие ковариаты помогают, но self-attention и LSTM требовательны к данным — отсюда отставание на коротком горизонте.

## Критичные гиперпараметры (из `run.py`)

| Параметр | Дефолт | Описание |
|---|---|---|
| `--past-window` | 72 | Длина входного окна. |
| `--d-model` / `--n-heads` | 128 / 4 | Размер модели / число голов. |
| `--lstm-layers` | 2 | Слои LSTM. |
| `--dropout` | 0.1 | Регуляризация. |
| `--epochs` | 20 | Число эпох. |
| `--batch-size` | 1024 | Размер батча. |
| `--learning-rate` / `--weight-decay` | 1e-3 / 1e-4 | Оптимизация. |
| `--synth-days` | 45 | Объём синтетики (`real_plus_synth`). |

## Ограничения

- Тяжелее наивных бейзлайнов, на 1 неделе данных не гарантирует преимущества (см. абляцию).
- На h=1 проигрывает графовым моделям по MAE/WAPE (выигрывает по SMAPE).
- Требует осмысленных будущих ковариат; качество зависит от их информативности.
