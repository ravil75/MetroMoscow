# N-BEATS (бейзлайн)

Код: [`src/models/nbeats/`](../../src/models/nbeats/) (`pipeline.py` + `run.py`). Замеры: строки `N-BEATS` в `metrics_tables.md`.

## Идея метода

N-BEATS — стек полносвязных блоков с **двойным остаточным разложением**: каждый блок выдаёт backcast (реконструкцию входа) и forecast (вклад в прогноз), а остаток передаётся дальше. В интерпретируемом варианте блоки специализируются на тренде (полиномы) и сезонности (гармоники Фурье).

## Как работает (высокий уровень)

- Вход — прошлое окно; стек блоков последовательно «вычитает» объяснённую часть (backcast) и накапливает forecast.
- `generic` — свободные базисы; `interpretable` — тренд (полином степени `--polynomial-degree`) + сезонность (`--num-harmonics`).
- Обучается глобально; опционально кондиционируется на будущие ковариаты (`--no-covariates` отключает).

## Применимость к малым данным

**Слабая на этой задаче.** По зрелым метрикам N-BEATS — самый слабый из глубоких на h=24 (MAE 61.77) и средний на h=1 (33.73). Чисто-MLP-стек с большим числом параметров склонен переобучаться на недельной истории; интерпретируемые базисы помогают, но не выводят его вперёд сезонных бейзлайнов на длинном горизонте.

## Критичные гиперпараметры (из `run.py`)

| Параметр | Дефолт | Описание |
|---|---|---|
| `--architecture` | generic | `generic` / `interpretable`. |
| `--num-stacks` | 3 | Число стеков. |
| `--num-blocks-per-stack` | 3 | Блоков на стек. |
| `--layer-sizes` | 256 | Ширина FC-слоёв. |
| `--polynomial-degree` | 3 | Степень тренда (interpretable). |
| `--num-harmonics` | auto (horizon//2) | Гармоники сезонности (interpretable). |
| `--past-window` | 72 | Длина входного окна. |
| `--epochs` / `--batch-size` | 15 / 2048 | Обучение. |

## Ограничения

- Большая параметрическая ёмкость → риск переобучения при дефиците данных.
- Базовый вариант не использует граф/связи между объектами.
- Интерпретируемые базисы требуют корректной настройки степени/гармоник под горизонт.
