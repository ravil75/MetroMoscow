# SAN — Slice Adaptive Normalization (идея 4)

Статья: **SAN** (NeurIPS’23). Флаг: `--san`, `--san-period` (дефолт 24). Замеры: конфиг **D + SAN** в `metrics_tables.md`.

## Идея метода

Нестационарность ряда (дрейф уровня, меняющаяся дисперсия) мешает модели. SAN режет ряд на **слайсы** (по умолчанию суточные) и **нормирует каждый слайс отдельно**, а затем отдельной головой предсказывает статистики (среднее/масштаб) будущего слайса, чтобы вернуть прогноз в исходный масштаб.

## Как работает (высокий уровень)

1. Вход делится на слайсы длиной `--san-period` (24 ч).
2. Каждый слайс нормируется по своим статистикам → модель работает со «стационаризованным» сигналом.
3. Лёгкая голова предсказывает статистики будущего слайса.
4. Прогноз денормируется обратно предсказанными статистиками.

## Применимость к малым данным

**Нейтрально (⚖).** По замерам D + SAN ≈ чистый D: на h=1 косметический плюс по MAE/WAPE, на h=24 лёгкий минус. Причина — в EgoGAT уже есть **seasonal-naive якорь** (вычитание профиля час × будни/выходной), который снимает уровень и суточную сезонность; послайсовая нормировка SAN поверх этого **избыточна** и нового сигнала почти не даёт. То есть на этих данных борьба с нестационарностью уже решена якорем.

## Критичные гиперпараметры

- `--san-period` (24) — длина слайса; суточный цикл — естественный выбор для часовых данных.
- Совместимость: применяется поверх конфига D (обе идеи).

## Ограничения

- Избыточна при наличии сильного сезонного якоря/детрендинга (как здесь).
- Добавляет параметры (голова предсказания статистик) без выигрыша на малой выборке → на h=24 может слегка вредить.
- Полезность ожидается выше на данных с выраженным дрейфом уровня, которого в недельной выборке мало.
