# TCN — Temporal Convolutional Network (бейзлайн)

Код: [`src/models/tcn/`](../../src/models/tcn/) (`pipeline.py` + `run.py`). Замеры: строки `TCN` в `metrics_tables.md`.

## Идея метода

TCN — свёрточная сеть для последовательностей: стек **дилатированных каузальных свёрток**, у которого рецептивное поле растёт экспоненциально с числом слоёв. Каузальность (свёртка смотрит только в прошлое) исключает утечку будущего.

## Как работает (высокий уровень)

- Несколько уровней (`--levels`) 1D-свёрток с растущей дилатацией покрывают длинный контекст за немного слоёв.
- Остаточные связи стабилизируют обучение.
- Обучается глобально, батч по (объект, окно); прогноз на горизонт 1 ч / 24 ч.

## Применимость к малым данным

**Слабее-средняя.** По зрелым метрикам TCN уступает TFT и сильным GAT-конфигам на обоих горизонтах (h=1 MAE 37.92, h=24 MAE 57.65). Свёрточная модель компактна и быстра, но без сезонного якоря/графа на недельной истории не выигрывает у наивных сезонных бейзлайнов по форме прогноза.

## Критичные гиперпараметры (из `run.py`)

| Параметр | Дефолт | Описание |
|---|---|---|
| `--levels` | 4 | Число дилатированных уровней (рецептивное поле). |
| `--hidden-channels` | 64 | Ширина свёрточных каналов. |
| `--kernel-size` | 3 | Размер ядра. |
| `--past-window` | 72 | Длина входного окна. |
| `--dropout` | 0.10 | Регуляризация. |
| `--epochs` | 15 | Число эпох. |
| `--batch-size` | 512 | Размер батча. |
| `--protocol` | fast | По умолчанию быстрый протокол (есть и rolling). |

## Ограничения

- Нет явной сезонной/графовой индуктивной структуры → на коротких данных проигрывает.
- Рецептивное поле ограничено `levels`/`kernel-size`; недельную сезонность ловит хуже, чем 2D-подходы.
- Чувствителен к подбору числа уровней и ширины каналов при дефиците данных.
