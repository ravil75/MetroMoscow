# TimesNet (идея 5)

Статья: **TimesNet** (ICLR’23, arXiv 2210.02186). Флаги: `--timesnet`, `--timesnet-blocks`, `--timesnet-k`. Замеры: конфиг **D + TimesNet (blocks 1)** в `metrics_tables.md`.

## Идея метода

Временной ряд содержит несколько периодичностей (суточная, недельная). TimesNet находит доминирующие периоды через FFT и **разворачивает 1D-ряд в 2D-тензор** (период × число периодов), после чего обрабатывает его 2D-свёртками. Так внутри- и меж-периодные зависимости моделируются как пространственная структура.

## Как работает (высокий уровень)

1. FFT определяет top-k периодов (`--timesnet-k`).
2. Ряд решейпится в 2D по каждому периоду.
3. 2D-свёрточные блоки (`--timesnet-blocks` штук) извлекают признаки.
4. Результаты по периодам агрегируются обратно в 1D-представление.

В EgoGAT TimesNet **заменяет** dilated-causal темпоральный энкодер.

## Применимость к малым данным

**Полезна точечно.** Единственная идея, давшая буст поверх D. По зрелым метрикам она **смещает выигрыш в процентную/объёмную точность**: лучший SMAPE на h=1 (18.57) и лучший WAPE на h=24 (20.37). По MAE на зрелых фолдах не превосходит чистый D (3-фолдовый «чемпионский» MAE 24.94/41.87 размывается на последних 2 фолдах). 2D-структура суток особенно ценна на длинном горизонте.

## Критичные гиперпараметры

| Параметр | Дефолт | Замечание |
|---|---|---|
| `--timesnet-blocks` | 2 | На h=1 нужен **1** — иначе не влезает в лимит Kaggle (TimesNet ~4.6× медленнее dilated). |
| `--timesnet-k` | 2 | Число топ-периодов FFT. |

Дополнительно: FFT под AMP/fp16 падает (cuFFT half только для размеров-степеней-2, T=72 — нет) → считается в fp32 (учтено в коде).

## Ограничения

- **Дорогой по времени** (~4.6× dilated) — главный ограничитель на h=1.
- Размен метрик: улучшает SMAPE/WAPE, но не MAE на зрелом срезе.
- Взаимоисключим с TimeMixer (другой альтернативный энкодер).
