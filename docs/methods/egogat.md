# EgoGAT (графовая модель, целевая)

Код: [`src/models/gat/pipeline.py`](../../src/models/gat/pipeline.py), CLI [`run.py`](../../src/models/gat/run.py). Конфиги абляции — см. README (A/B/C/D, D+SAN, D+TimesNet).

## Идея метода

EgoGAT — графовая attention-модель для **global forecasting**: один набор весов прогнозирует все ~1500 объектов сразу, используя не только собственную историю объекта, но и историю его **соседей по графу** (пересадки, узлы, скоррелированные станции). Ключевая идея — заменить full-graph батчинг на батчинг по **(объект, окно)** и строить «эго-граф» вокруг каждого объекта.

## Как работает (высокий уровень)

- **Граф (ego).** Для каждого объекта берутся top-k соседей по корреляции **остатков** (из ряда вычитается профиль час × будни/выходной), что ловит совместные отклонения, а не общий суточный цикл (`build_graph`).
- **Темпоральный энкодер.** Dilated causal conv сохраняет все T состояний истории (по умолчанию; альтернативы — TimesNet/TimeMixer).
- **Декодер.** Per-step GRU + temporal cross-attention на собственную историю + gated spatial cross-attention на состояния соседей (выражает lead-lag между объектами).
- **Выход** = seasonal-naive якорь + обучаемая поправка; ночные часы даунвешиваются в лоссе.
- **FiLM-кондиционирование** на статические признаки узла (профиль, объём, кластер).
- **Идеи из статей** включаются флагами: adaptive embeddings, adaptive adjacency, SAN, TimesNet (отдельные файлы в этой папке).

## Применимость к малым данным

**Подходит.** Несколько механизмов прямо адресуют low-data regime:

- Батчинг по (объект, окно) даёт тысячи i.i.d. шагов оптимизатора из одной недели данных.
- Глобальная модель (общие веса на все объекты) — это сильный pooling: редкие паттерны одного объекта подкрепляются похожими объектами.
- Seasonal-naive якорь означает, что модель учит лишь **поправку** к сильному эвристическому прогнозу — меньше параметров «с нуля».
- Граф регуляризует: сосед даёт дополнительный сигнал, снижая переобучение на коротком ряде.

**Риск:** обучаемые компоненты (особенно эмбеддинги узлов) могут переобучаться — что и видно в абляции на h=24 (см. adaptive_embeddings.md).

## Критичные гиперпараметры

| Параметр | Дефолт | Замечание |
|---|---|---|
| `--past-window` | 72 | **Не** ставить 168 (ломает h=24, MAE ~103). |
| `--step-1h` | 6 | **Не** ставить 24 (ломает h=1). |
| `--top-k-neighbors` | 8 | Число соседей в эго-графе. |
| `--min-corr` | 0.05 | Порог корреляции остатков для ребра. |
| `--night-weight` | 0.3 | Вес ночных часов (0–5) в лоссе. |
| `--d-model` / `--n-heads` | 128 / 4 | Размер модели. |
| `--epochs` | 11–12 | Короткая история → немного эпох. |
| `--synth-days` | 45 | Объём синтетики (`real_plus_synth`). |

## Ограничения

- Граф строится по **корреляции остатков**, отдельной таблицы физических связей (пересадок) в данных нет.
- TimesNet ~4.6× медленнее dilated — на h=1 нужен `--timesnet-blocks 1`, иначе не влезает в лимит Kaggle (~12 ч).
- На h=24 «голый» baseline (без идей) уступает TFT — выигрыш даёт только пара adaptive-идей (D) и TimesNet.
- Метрики чувствительны к мелким низкопоточным объектам (раздувают SMAPE/MAPE).
